###angular简介
angular是一款非常优秀的前端高级框架，其核心通过指令扩展了html，通过
表达式绑定数据到html。在angular中是不推荐使用dom操作的，因为angular倡导
以测试驱动开发，在service或controller中出现了DOM操作，那么就意味着测试是
无法通过的。使用框架可以轻松实现SPA(单页应用)应用程序。

###angular安装
1.暴力安装：直接复制一个angular.js
2.npm方式：npm install angular@1.5.2安装指定版本的
3.bower 方式 ‘npm install angular’

###指令
1.ng-app指令告诉angularjs应用当前这个元素是根元素，所有angularjs都
必须有一个根元素，当有多个ng-app指令，只有第一个会使用
2.ng-model把对应的值显示到页面中：例ng-model=”var”
3.ng-click用来注册点击事件：例ng-click=”add()”
4.ng-init用于初始化某个变量：例ng-init=”var= ‘小明’”

###将ng-model值显示到页面中

###什么是MVC思想和MVVM思想
M：Model 模型：数据存储，一些业务逻辑
V：View 视图 ：用来展示数据
C Controller 控制器：调度业务逻辑
VM ViewModel

###angular核心特性
1.模块化 angular.module()
angular.module(‘myApp’,[])
第一个参数是模块的名字
第二个是一个数组，数组的元素是该模块所依赖的其他模块的名字，若无依赖，传空数组
2.控制器controller
angular.moule(‘myApp’,[]).controller(‘myController’,function($scope){}));
第一个参数是控制器的名字
第二个参数是一个回调函数，传入参数$scope($scope对象在angularjs中充当数据模型的作用，也就是一般MVC框架中Model的角色，但与通常意义上的数据模型不一样，
因为$scope并不处理和操作数据，他只是建立了视图和html之间的桥梁，让视图和controller之间可以友好的通讯)
2.双向数据绑定
数据模型的值发生改变，就会导致页面值的改变，页面值的改变就会导致数据模型值
的改变，这种相互影响的关系叫双向数据绑定。

canvas

##canvas是一个画布，默认大小是300*150

如果想改变大小必须通过html的width，height属性增加像素改变大小，不能直接通过css设置，css设置会直接拉伸像素大小，并不能改变画布大小

##想要在canvas上绘图，必须得到canvas绘图工具

var ctx = canvas.getContext("2d");   2d 平面绘图    webgl 3d绘图
ctx.moveTo(100,100);//将绘图工具定点到100,100位置
ctx.lineTo(200,100);//从刚才位置绘制到哪里
ctx.stroke();描边绘图，将绘制的图形呈递出来

##ctx.fill()

调用fill方法，会将所有的点连接起来，即使没有结束的那条线，也会构成一个封闭的结构

###ctx.closePath() 

闭合路径方法，可以将结束点与开始点连接起来
ctx.lineWidth = num：用来设置线条的宽度  num是宽度

###ctx.beginPath() 

在canvas中开启新的内容，不影响上一次绘制

###虚线

ctx.setLineDash(arr)   arr中存在着实线部分和虚线部分

###颜色

ctx.strokeStyle= 'red';  设置描边颜色
ctx.fillStyle= 'red';  设置填充颜色

##绘制矩形

如果不设置style，不需要设置beginPath
ctx.rect(x,y,width,height);   虚描，需要描边  x,y起始点
ctx.strokeRect(x,y,width,height);   直接画矩形
ctx.fillRect(x,y,width,height);   填充矩形
ctx.clearRect(x,y,width,height); 清除绘制的矩形区域,橡皮擦功能
ctx.clearRect(0,0,cas.width,cas.height);清除整个画布
cas.width = cas.width;  相当于把画布刷新，也可以清除整个画布

##绘制圆弧

ctx.arc(x,y,r,startAngle,endAngle,clockWise);   需要描边呈现
x,y圆心坐标;r圆弧半径；startAngle圆开始角度；endAngle圆结束角度(需要转换为弧度) ；Math.PI = 180度   radian = angle * Math.PI /180；

##绘制文本

ctx.strokeText(文本，x,y);  文本字符串的左下角在x,y坐标处；文本内容空心
ctx.fillText(文本，x,y);   实心文本内容
ctx.font = '30px 黑体';   设置字体样式  大小、字体
ctx.textAlign水平对齐 取值left center right ； start end 默认为start  
ctx.textBaseline基线对齐 取值 top  middle bottom alphabetic(基线对齐)
ctx.measureText(文本).width；获取当前文本的宽度

##图片绘制

1.创建图片对象  var  img = new Image(); 
2.img.src = '...'
3.1 ctx.drawImage(img,x,y);  将图片绘制到x,y的位置
3.2 ctx.drawImage(img,x,y,width,height); 将图片绘制到矩形区域，
进行计算，等比缩放，
3.3 ctx.drawImage(img,sx,sy,sw,sh,x,y,w,h); 将图片的某个部分
绘制到canvas中，一般截取的部分宽高与绘制的宽高一致
drawImage不仅可以绘制图片，也可以绘制其他其它标签，例video、canvas等

##变换

变换操作的是坐标轴，当坐标轴发生了变化以后，坐标轴内的图形也会跟着变化
ctx.translate()  平移变换
ctx.rotate()  旋转变换
ctx.scale()   伸缩变换

##canvas状态

canvas中，凡是设置属性，都会延续到下一次修改。
ctx.save()  将当前状态保存   代码开始使用
ctx.restore()  将保存的状态恢复   代码结束使用
save 和 restore搭配使用，可以使上一次操作不影响下一次的

    git将本地文件push到远端仓库

    #（先进入项目文件夹）通过命令 git init 把这个目录变成git可以管理的仓库
    git init

    #把文件添加到版本库中，使用命令 git add .添加到暂存区里面去，不要忘记后面的小数点“.”，意为添加文件夹下的所有文件
    git add .
    如出问题:git warning: LF will be replaced by CRLF in 解决办法
    git config core.autocrlf false

    #用命令 git commit告诉Git，把文件提交到仓库。引号内为提交说明
    git commit -m ‘first commit’

    #关联到远程库
    git remote add origin 你的远程库地址

    #获取远程库与本地同步合并（如果远程库不为空必须做这一步，否则后面的提交会失败）
    git pull –rebase origin master

    #把本地库的内容推送到远程，使用 git push命令，实际上是把当前分支master推送到远程。执行此命令后会要求输入用户名、密码，验证通过后即开始上传。

    gulp

    ##使用npm安装指定版本的angular
    npm install angular@1.4.7 可以安装指定版本为1.4.7的文件，@后跟版本号

    ##在gulpfile.js文件中配置gulp
    所有引用的gulp插件以及gulp都需要使用npm进行局部安装，然后在根目录创建
    gulpfile.js，在文件中配置任务，在本文件夹下打开的命令窗口输入gulp [任务名]
    例：gulp script 就可以执行名称为script的任务

    ###1.引入gulp模块
    var gulp = require(‘gulp’);

    ###2.引入其他gulp插件
    1.//引入js压缩插件gulp-uglify
    var uglify = require(‘gulp-uglify’);
    2.//引入css压缩插件gulp-cssnano
    var cssnano = require(‘gulp-cssnano’);
    3.//引入html压缩插件
    var htmlmin = require(‘gulp-htmlmin’);
    4.//引入合并插件gulp-concat
    var concat = require(‘gulp-concat’);
    5.//引入文件重命名文件gulp-rename
    var rename = require(‘gulp-rename’);
    6.//引入js代码错误提醒插件
    var jshint = require(‘gulp-jshint’);
    7.//引入文件重命名插件
    var imagemin = require(‘gulp-imagemin’);
    8.//引入更改提醒插件
    var notify = require(‘gulp-notify’);
    9.//引入浏览器自动刷新插件
    var browserSync = require(‘browser-sync’).create();
    var reload = browserSync.reload;
    其它插件：用法见https://www.npmjs.com/search?q=gulp
    sass的编译（gulp-sass）
    less编译 （gulp-less）
    自动添加css前缀（gulp-autoprefixer）
    压缩图片（gulp-imagemin）
    自动刷新页面（gulp-livereload，谷歌浏览器亲测，谷歌浏览器需安装livereload插件）
    图片缓存，只有图片替换了才压缩（gulp-cache）

    ###创建任务
    1.html压缩 htmlmin使用
    gulp.task(‘html’,function(){ //task方法是创建任务的实现方式
    gulp.src(‘./index.html’)
    //src方法获取文件，此处获取同文件夹下的index，html文件
    .pipe(htmlmin({collapseWhitespace:true})) //去除html中的空白符
    .pipe(gulp.dest(‘./dist’))
    //dest方法，指定压缩后的文件存放路径，次例：如果没有dist文件夹会自动创建
    .pipe(notify(‘hello html’));//任务完成后提醒插件使用
    });
    2.rename实例 更改html的名称
    gulp.task(‘rename’,function(){
    gulp.src(‘./index.html’) //获取文件
    .pipe(rename({
    direname:”./“, //所在目录
    basename:”aloha”, //
    prefix:”bonjour-“, //
    suffix:”-hola”, //
    extname:”.html” // 后缀名
    })) //效果生成bonjour-aloha-hola.html文件名
    .pipe(gulp.dest(‘./‘)); //更改后文件存放在本目录
    });
    3.css压缩
    gulp.task(‘styles’,function(){
    gulp.src(‘./css/.css’) //选择所有css目录下的css文件
    .pipe(cssnano()) // 压缩css代码
    .pipe(gulp.dest(‘./dist/css’)) // 压缩后保存目录
    .pipe(browserSync.stream()) // css改变后，浏览器能收到修改信息
    .pipe(notify(‘css complete’)) // 任务完成提醒
    });
    4.js语法检测、代码合并、压缩
    gulp.task(‘scripts’, function() {
    //合并操作的方法，此处不执行
    //gulp.src([‘./demo.js’,’./index.js’])//要合并操作的文件
    //.pipe(concat(‘all.js’)) //合并后名称
    gulp.src(‘js/    .js’)
    .pipe(jshint()) // 进行检查
    .pipe(jshint.reporter(‘default’)) // 对代码进行报错提示
    .pipe(uglify()) //js代码压缩
    .pipe(gulp.dest(‘./dist/js’))
    .pipe(browserSync.stream()); //js代码更新，浏览器能收到修改信息
    });
    5.图片:压缩
    gulp.task(‘images’, function() {
    gulp.src(‘./images/.‘) //选的images文件夹下的所有图片
    .pipe(imagemin())
    .pipe(gulp.dest(‘dist/images’))
    });
    6.主任务
    // 主任务
    gulp.task(‘default’, [‘html’, ‘styles’, ‘scripts’, ‘images’], function() {
    //在执行default任务时，先执行[‘html’, ‘styles’, ‘scripts’, ‘images’]
    //的任务，然后再执行下面的代码(称之为任务依赖)
    browserSync.init({ //初始化浏览器自动更新操作
    server: “./dist” //要管理的文件属于哪个文件夹
    });
    //当指定的文件发送改变时执行指定任务
    gulp.watch(‘./index.html’, [‘html’]);
    gulp.watch(‘./css/.css’, [‘styles’]);
    gulp.watch(‘./js/    .js’, [‘scripts’]);
    //当任意html文件发送变化刷新浏览器
    gulp.watch(“./*.html”).on(‘change’, browserSync.reload);
    });

    gulp配置信息

    ###配置gulp

    ####先安装好各个依赖的插件
    1.首先安装node环境，安装gulp插件使用npm进行
    2.进入工作目录，在此处打开命令行
    3.使用npm管理该文件夹，npm init -y 快速生成package.json文件
    4.通过npm安装各个gulp的插件
    npm install gulp –save-dev //基础库
    npm install gulp-htmlmin –save-dev //html压缩
    npm install gulp-csso –save-dev //css压缩
    npm install jshint gulp-jshint –save-dev //js检查
    npm install gulp-uglify –save-dev //js压缩
    npm install gulp-concat –save-dev //文件合并
    npm install gulp-clean –save-dev //清空文件夹
    npm install gulp-imagemin –save-dev //图片压缩
    npm install gulp-rename –save-dev //文件重命名
    npm install gulp-rev –save-dev //更改版本名
    npm install gulp-rev-collector –save-dev
    //gulp-rev的插件，用于HTML模板更改引用路径

    ####在当前文件夹的根目录下建立gulpfile.js文件，进行配置
    1.首先将各个依赖的插件引入
    var gulp = require(‘gulp’), //基础库
    htmlmin = require(‘gulp-htmlmin’),//html压缩
    csso = require(‘gulp-csso’), //css压缩
    jshint = require(‘gulp-jshint’), //js检查
    uglify = require(‘gulp-uglify’), //js压缩
    concat = require(‘gulp-concat’), //文件合并
    clean = require(‘gulp-clean’), //清空文件夹
    imagemin = require(‘gulp-imagemin’), //图片压缩
    rename = require(‘gulp-rename’), //文件重命名
    rev = require(‘gulp-rev’), //更改版本名
    revCollector = require(‘gulp-rev-collector’); //gulp-rev的插件，用于HTML模板更改引用路径
    2.开始配置任务
    // 1.html压缩 htmlmin使用
    gulp.task(‘html’, function () { //task方法是创建任务的实现方式
    var options = {
    collapseWhitespace:true,
    collapseBooleanAttributes:true,
    removeComments:true,
    removeEmptyAttributes:true,
    removeScriptTypeAttributes:true,
    removeStyleLinkTypeAttributes:true,
    minifyJS:true,
    minifyCSS:true
    };
    return gulp.src(‘index.html’)
    //src方法获取文件，此处获取同文件夹下的index，html文件
    .pipe(htmlmin(options)) //去除html中的空白符
    .pipe(gulp.dest(‘dist/‘));
    //dest方法，指定压缩后的文件存放路径，此时：如果没有dist文件夹会自动创建
    });
    //2.css压缩
    gulp.task(‘styles’, function () {
    return gulp.src(‘./css/.css’) //选择所有css目录下的css文件
    .pipe(csso()) // 压缩css代码
    // 此处不要修改文件名称，不然下面rev任务自动替换引用文件时会找不到要替换的内容
    // .pipe(rename(function (path) {
    // path.basename += ‘.min’;
    // path.extname = ‘.css’
    // }))
    .pipe(rev())
    .pipe(gulp.dest(‘./dist/css’))// 压缩后保存目录
    .pipe(rev.manifest())
    .pipe(gulp.dest(‘./rev/css’));
    });
    // 3.js语法检测、代码合并、压缩
    gulp.task(‘scripts’, function () {
    //合并操作的方法，此处不执行
    //gulp.src([‘./demo.js’,’./index.js’])//要合并操作的文件
    //.pipe(concat(‘all.js’)) //合并后名称
    return gulp.src(‘./js/    .js’)
    .pipe(jshint()) // 进行检查
    .pipe(jshint.reporter(‘default’)) // 对代码进行报错提示
    .pipe(uglify()) //js代码压缩
    .pipe(rev())
    .pipe(gulp.dest(‘./dist/js’))// 压缩后保存目录
    .pipe(rev.manifest())
    .pipe(gulp.dest(‘./rev/js’));
    });
    // 4.图片:压缩
    gulp.task(‘images’, function() {
    gulp.src(‘./images/.‘) //选的images文件夹下的所有图片
    .pipe(imagemin())
    .pipe(gulp.dest(‘dist/images’))
    });
    //5. rev使用，自动改变HTML文件中的引用路径
    gulp.task(‘rev’,function () {
    return gulp.src([‘./rev/*/.json’,’./dist/index.html’])
    .pipe(revCollector({
    replaceReved:true
    }))
    .pipe(rename(function (path) {
    path.basename += ‘.min’;
    path.extname = ‘.html’
    }))
    .pipe(gulp.dest(‘./dist’));
    });
    // 主任务，只需在命令行执行此任务就能实现压缩代码，rev任务除外
    gulp.task(‘default’, [‘html’, ‘styles’, ‘scripts’,’images’, ‘rev’], function () {
    //在执行default任务时，先执行[‘html’, ‘styles’, ‘scripts’, ‘images’]
    //的任务，然后再执行下面的代码(称之为任务依赖)
    //当指定的文件发送改变时执行指定任务
    gulp.watch(‘./index.html’, [‘html’]);
    gulp.watch(‘./css/.css’, [‘styles’]);
    gulp.watch(‘./js/    .js’, [‘scripts’]);
    });

    ###注意：在执行完default主任务后没有实现HTML中路径自动改变，需要单独执行rev任务才能为HTML引入的文件自动加上版本号
